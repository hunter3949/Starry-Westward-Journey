### 2-2. process_checkin RPC（原子性打卡）

星光版將打卡邏輯從 TypeScript 移到 PostgreSQL RPC（571 行），好處：
- 資料庫層級 `FOR UPDATE` 鎖定，避免並發重複打卡
- 所有計算（法寶加成、等級提升、屬性成長、金幣）在一個 transaction 內完成
- 比巨笑版的 `quest.ts`（分段讀寫）更安全

包含三個函數：
- `calculate_level_from_exp(p_exp)` — 等級計算
- `process_checkin(p_user_id, p_quest_id, ...)` — 原子打卡
- `process_undo(p_user_id, p_quest_id, ...)` — 復原打卡

**步驟**：
1. 複製 migration `202604040001_process_checkin_rpc.sql`
2. 調整欄位名稱（星光版用 `SquadName`，巨笑版用 `LittleTeamLeagelName`）
3. 修改 `quest.ts` 改為呼叫 RPC

```
來源：example/supabase/migrations/202604040001_process_checkin_rpc.sql
目標：新增 migration + 修改 app/actions/quest.ts
```

4. **GameGold 分離**：戰鬥獎勵改用 `GameGold` 後，需確認商店 Tab 的顯示邏輯
5. **型別更新**：`types/index.ts` 需加入 `IsBlessed`、`DemonDropBoostSeasonal`、`d7_activated_at` 等欄位
