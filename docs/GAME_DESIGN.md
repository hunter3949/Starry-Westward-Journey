## 一、遊戲核心理念

《大無限開運西遊》以西遊記為世界觀，將「五毒對治」（嗔、貪、痴、慢、疑）轉化為遊戲機制。玩家每日完成「親證定課」便能對治角色的五毒詛咒，解鎖地圖特殊免疫，並在六角格地圖上與隊友協力冒險。

### 2.2 各角色詳細設定

#### 🐒 孫悟空（嗔區守護者）
- **被動天賦：越戰越勇**：連續打卡可疊加攻擊力，可無視迷霧陷阱。
- **五毒詛咒：緊箍咒**
  - 觸發條件：當日 DailyLogs 未包含 `q2` 感恩冥想。
  - 效果：「暴躁」狀態。下次移動路徑發生隨機偏移，防禦力 -30%。
- **地圖免疫（嗔區）**：完成 `q1` 打拳 **或** `q2` 感恩冥想，獲得「清涼心」Buff，免疫熔岩扣血。
- **等級成長**：Physique +3, Potential +2, Charisma +1, Luck +1, Savvy +1（每升一級）

#### 豬（豬八戒）（貪區守護者）
- **被動天賦：福星高照**：資源雙倍，骰子滿格可恢復 HP。技能「九齒釘耙」可挖掘寶物。
- **五毒詛咒：貪吃誤事**
  - 觸發條件：當日同時缺乏 `q6` 海鮮素 **和** `q7` 子時入睡。
  - 效果：「懶惰」狀態。移動每格額外消耗 0.5 AP，當天骰子強制歸零。
- **地圖免疫（貪區）**：完成 `q6` **或** `q7`，獲得「輕盈」Buff，深淵泥淖不強制停止。
- **等級成長**：Luck +4, Charisma +1, Potential +1, Spirit +1, Savvy +1

#### 🐢 沙悟淨（痴區守護者）
- **被動天賦：捲簾大將**
  - 相鄰隊友雙方防禦 +10%。
  - 通過「西南痴區迷走流沙」地形時，免疫強制位移與方向錯亂。
- **五毒詛咒：迷霧障眼**
  - 觸發條件：當日 DailyLogs 未包含 `q4` 嗯啊吽七次。
  - 效果：「無明」狀態。地圖怪物等級與數值全部隱藏。
- **地圖免疫（痴區）**：完成 `q4`，獲得「定心」Buff，免疫流沙位移與方向錯亂。
- **等級成長**：Spirit +2, Physique +2, Potential +2, Charisma +1, Luck +1

#### 🐎 白龍馬（慢區守護者）
- **被動天賦：日行千里**：移動骰基礎 +2，可回收剩餘移動力。技能「龍騰」。
- **五毒詛咒：傲慢之牆**
  - 觸發條件：當日 DailyLogs 未包含 `q5` 五感恩。
  - 效果：「孤立」狀態。無法接受或給予團隊 Buff。
- **地圖免疫（慢區）**：完成 `q5`，獲得「謙卑」Buff（待定）。
- **等級成長**：Charisma +3, Luck +3, Savvy +1, Spirit +1

#### 🧘 唐三藏（疑區守護者）
- **被動天賦：信念之光**：加成傳愛獎勵，可施放範圍回血。技能「般若咒」範圍傷害。
- **五毒詛咒：寸步難行**
  - 觸發條件：當日未完成 `q3` 當下之舞 **或** `CharacterStats.Streak = 0`。
  - 效果：「自我懷疑」狀態。無法自行擲骰，消耗雙倍能量骰子才能行動，或等待隊友「拉一把」。
- **地圖免疫（疑區）**：完成 `q3`，獲得「心燈」Buff，周圍迷霧自動散去。
- **等級成長**：Spirit +4, Potential +3, Savvy +1

## 三、定課與任務系統 (Quest System)

### 3.1 每日定課（Daily Quests）

每日最多完成 **3 項** `q` 開頭的定課。

| ID | 名稱 | 說明 | 經驗 | 骰子 |
|----|------|------|------|------|
| `q1` | 打拳 | 身體開發（一天中任何時段完成均算） | 200 | 1 |
| `q2` | 感恩冥想 | 對治嗔心 | 100 | 1 |
| `q3` | 當下之舞 | 對治疑心 | 100 | 1 |
| `q4` | 嗯啊吽七次 | 覺醒痴念 | 100 | 1 |
| `q5` | 五感恩 | 放下傲慢 | 100 | 1 |
| `q6` | 海鮮素 | 節制貪慾 | 100 | 1 |
| `q7` | 子時入睡 | 能量補給 | 100 | 1 |
| `q8` | 高階定課1 | 進階修行 | 100 | 1 |
| `q9` | 高階定課2 | 進階修行 | 100 | 1 |

#### 打拳（q1）的兩種模式

```
┌────────────────────────────────────────┐
│  打拳定課（q1）打卡                      │
│                                        │
│  🥊 一般打拳（預設）                    │
│     一天中任何時段完成 → +200 修為       │
│                                        │
│  🌅 破曉打拳（勾選選項）                │
│     太陽剛升起時段（05:00–08:00）完成    │
│     → +200 修為（基礎）                 │
│     + 持有照妖鏡(a2) → 額外 +150 修為  │
└────────────────────────────────────────┘
```

- 打卡 UI 在 `q1` 打拳的回報按鈕下方，新增一個勾選選項：**「本次為破曉打拳（05:00–08:00 完成）」**
- 勾選後系統以 `q1_dawn` 作為 QuestID 寫入記錄，與一般 `q1` 分開識別
- 持有 `a2 照妖鏡` 且勾選「破曉打拳」者，打卡時自動觸發 **+150 修為** 加成
- 無照妖鏡者勾選破曉選項後，僅記錄為「破曉打拳」但無額外加成（效果等同一般打拳）



### 3.2 每週活動（Weekly Quests）

| ID | 名稱 | 說明 | 經驗 | 上限 | 骰子加成 |
|----|------|------|------|------|---------|
| `w1` | 小天使通話 | 關心夥伴 (15min) | 500 | 1次 | +1 骰子 |
| `w2` | 參加心成活動 | 聚會/培訓/活動 | 500 | 2次 | +2 骰子 |
| `w3` | 家人互動親證 | 視訊或品質陪伴 | 500 | 1次 | +1 骰子 |
| `w4` | 傳愛分數 | 訪談成功加分 | 1000 | 無上限 | +1 骰子 |

#### `w4` 傳愛分數審核流程

> 防止自行填報，所有傳愛分數需經**三級核實**後才納入經驗計算。

```
隊員提出申請
    ↓
小隊長確認（同小隊層級驗證）
    ↓
（大）隊長複核
    ↓
大隊長核實 ✅ → 系統寫入 DailyLogs，經驗生效
             ❌ → 退回，不計分
```

| 角色 | 職責 |
|------|------|
| 隊員 | 填寫申請（訪談對象、日期、簡述） |
| 小隊長 | 初審，確認成員當週活動狀況 |
| 大隊長 | 終審核實，核准後觸發系統入帳 |

### 3.3 體系活動 (`t` 開頭)
- 主題親證、會長交接、大會等：完成可獲得 **1 黃金骰子**。
- 搭配神器 `a4 幌金繩`，體系活動個人經驗 ×1.5。

### 3.4 特殊仙緣任務（臨時加分副本）

> 由管理者透過後台即時建立、上下架，玩家端顯示為限時加分機會，不計入每日定課上限（3 次）。

#### 玩家端顯示格式

```
┌─────────────────────────────────────┐
│  ✨ 特殊仙緣任務                      │  ← 主標題（由 title 欄位控制，管理者自訂）
│  跟父母三道菜                         │  ← 任務名稱（sub，橘色加粗顯示）
│  面對面或是視訊均可                   │  ← 任務說明（desc，灰色斜體）
│  +500 修為                           │  ← 獎勵經驗
└─────────────────────────────────────┘
```

#### 資料結構（`TemporaryQuest`）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | `string` | 系統自動生成 UUID |
| `title` | `string` | 任務主標題（管理者輸入，如「春節家庭任務」） |
| `sub` | `string` | **任務名稱**（短標籤，如「跟父母三道菜」），橘色加粗顯示 |
| `desc` | `string?` | **任務說明**（完成標準，如「面對面或是視訊均可」），灰色斜體 |
| `reward` | `number` | 加分額度（經驗值），預設 500 |
| `active` | `boolean` | 是否啟用中（控制玩家端可見性） |

#### 後台管理操作（`AdminDashboard` → 臨時加分任務管理）

| 動作 | 函式 | 說明 |
|------|------|------|
| 新增 | `onAddTempQuest(title, sub, desc, reward)` | 依序填入主標、任務名稱、說明、加分額度 |
| 啟用 / 暫停 | `onToggleTempQuest(id, active)` | 🟢 啟用中 / 🔴 已暫停，即時切換 |
| 刪除 | `onDeleteTempQuest(id)` | 永久移除（無法復原） |

#### 限制與規則

- 無數量上限，管理者可同時開啟多個特殊仙緣任務
- 每位玩家**每個任務 ID 只能領取一次**（防刷）
- `active = false` 時任務在玩家端隱藏，但資料保留
- 不設截止時間，由管理者手動下架

### 3.5 每週推薦定課（小隊長抽籤制）

> **v0.4 修訂**：廢除管理員統一指定制與每日缺課罰金機制，改為小隊長每週隨機抽籤。

#### 抽籤規則

| 項目 | 說明 |
|------|------|
| **抽籤時機** | 每週一中午 **12:00 前**，由小隊長在「指揮所」頁面手動執行 |
| **抽籤範圍** | `q1`~`q7` 共 7 項日常定課，本週推薦的定課由此隨機選出 |
| **不重複規則** | 第二週起，已抽過的定課不再出現，直到 7 項全部抽完後重置循環 |
| **逾時自動補抽** | 若小隊長週一 12:00 前未抽，管理員可在後台觸發「全服自動抽籤」，為所有未抽小隊代為隨機選出 |
| **顯示位置** | 玩家「修行定課」頁最上方顯示「本週推薦定課」卡片（不含罰金） |

#### DB 欄位（`TeamSettings`）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `mandatory_quest_id` | `text` | 本週抽出的定課 QuestID（複用舊欄位） |
| `mandatory_quest_week` | `text` | 本次抽籤的週一日期（`YYYY-MM-DD`），防止同週重複抽 |
| `quest_draw_history` | `jsonb` | 已抽過的 QuestID 陣列，用於不重複邏輯；全部抽完後清空重置 |

#### 抽籤 Server Actions（`app/actions/team.ts`）

| 函式 | 說明 |
|------|------|
| `drawWeeklyQuestForSquad(squadName, captainUserId)` | 小隊長手動抽籤，防重複抽同一週 |
| `autoDrawAllSquads()` | 管理員觸發，為本週尚未抽籤的小隊代為抽選 |

### 3.6 AI 功能總覽 — `gemini.ts`

所有 AI 功能使用 **Gemini 2.5 Flash**（`gemini-2.5-flash`），強制回傳 `responseMimeType: "application/json"`。

#### 3.6.1 AI 修行週報 — `generateWeeklyReview`

- **觸發時機**：玩家點擊「加分副本」Tab，系統自動判斷：若本週尚無週報則呼叫 Gemini 生成，已有則直接讀取（同週不重複呼叫）。
- **資料依據**：玩家本週 + 上週 `q` 開頭定課完成次數（上限 21 次/週），以及六維屬性最低項。
- **顯示位置**：「加分副本（Weekly）」Tab 最頂端，卡片形式顯示。
- **生成內容**：
  - `summary`（80–100 字）：個人化修行覆盤，語氣依完成率調整（高→鼓舞；中→提醒；低→當頭棒喝）。
  - `quote`（15–30 字）：西遊/道家主題金句，與本週狀態呼應。
  - `trend`：`"up" | "down" | "stable"`（完成率週差 > 5% 為 up/down）。
  - `weeklyRate`：本週完成率（0–1 小數）。
- **持久化**：結果寫入 `WeeklyReviews` 資料表，`UNIQUE(user_id, week_label)` 確保冪等（重跑安全）。
- **DB Schema**（`WeeklyReviews`）：

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | uuid | PK |
| `user_id` | text | 玩家 UserID |
| `week_label` | date | 當週週一日期（`YYYY-MM-DD`） |
| `content` | jsonb | `{ summary, quote, trend, weeklyRate }` |
| `created_at` | timestamptz | 生成時間 |

#### 3.6.2 AI 隊長建議 — `generateCaptainBriefing`

- **權限**：僅 `IsCaptain = true` 的玩家可使用。
- **觸發時機**：隊長在「隊長指揮所」Tab 手動點擊「🤖 開始分析」按鈕（即時生成，不快取）。
- **資料依據**：整支小隊（`TeamName`）所有成員的近 7 天 `q` 開頭定課完成次數，批量查詢（`ANY($1::text[])`）避免 N+1。
- **顯示位置**：「隊長指揮所」Tab 頂部，分析區塊 inline 展開（不開 modal）。
- **生成內容**：
  - `teamSummary`（80–100 字）：整體隊伍士氣評述，第二人稱「你的小隊」向隊長說話。
  - `topPerformer`：本週完成率最高的隊員姓名 + 一句鼓勵（格式：「姓名：鼓勵語」）。
  - `needsSupport`：完成率 < 33% 的隊員姓名陣列，無則為 `[]`。
  - `suggestion`（約 50 字）：針對小隊現況的本週具體行動建議。
  - `teamMorale`：`"high"（>70%）| "medium"（40–70%）| "low"（<40%）`。
- **無持久化**：結果為即時回傳，不寫入資料庫，每次點擊均重新生成。

## 四、屬性系統 (Stats System)

| 屬性 | 英文 | 說明 |
|------|------|------|
| 神識 | Spirit | 法術強度、精神防禦、抗異常狀態能力 |
| 根骨 | Physique | 體力、防禦、HP 上限 |
| 魅力 | Charisma | 社交觸發、團體加成接收率（NPC 互動機率） |
| 悟性 | Savvy | 技能效果、特殊判定 |
| 福緣 | Luck | 暴擊率、掉落率 |
| 潛力 | Potential | 戰鬥爆發力、等級屬性放大乘數（非自由配點） |
| 連續 | Streak | 連續打卡天數（唐三藏詛咒判定） |

### 戰鬥公式
- `baseATK = Level × 10 + Physique × 2`
- `baseDEF = roleConfig.baseDEF + Physique × 1`
- 暴擊率 = `Luck × 5%`，閃避率 = `max(0, (10 - Luck) × 2%)`

## 六、寶箱與偽裝寶箱系統 (Chest & Mimic)

### 6.1 基礎寶箱 (Chest)
觸發條件：踩到寶箱地格或點擊地圖寶箱圖示。（`handleChestOpen` in `map.ts`）

**掉落表**（`CHEST_LOOT_TABLE`）：
| 骰子數量 | 機率 |
|---------|------|
| 1 顆 | 60% |
| 2 顆 | 30% |
| 3 顆 | 10% |

### 6.2 偽裝寶箱（Mimic）
- **基礎觸發率**：20%（`MIMIC_CHANCE = 0.2`）
- 觸發後進行 **慧根 (Savvy) 檢定**：`d20 + Savvy vs DC 12`
  - **失敗**（骰點 + Savvy < 12）：扣除 **1 個能源骰子**
  - **成功**（骰點 + Savvy ≥ 12）：識破並反擊，獲得 **1 個能源骰子**
- 道具「照妖鏡 (i2)」可提前識破 Mimic，取消觸發。

### 6.3 黃金骰子「加持開箱」（`blessChestWithGoldenDice`）
- 消耗 **1 枚黃金骰子**，確保下一次開箱：
  - 必定獲得最高獎勵（3 顆骰子）
  - **無視** Mimic 判定
- 狀態以 `IsBlessed` 欄位記錄於 `CharacterStats`（待資料庫新增欄位）

## 七、戰鬥系統 (Combat System)

### 7.1 基本流程
1. 玩家使用剩餘 AP 進行連擊（每 1 AP = 1 次攻擊）。
2. 第一次攻擊享有戰術加成（側翼 ×1.3、背刺 ×1.5）。
3. 若怪物存活，反擊一次。
4. 戰鬥不提供經驗值，僅獎勵**遊戲黃金 (GameGold)** 與骰子。

### 7.2 戰鬥獎勵
| 類型 | 計算 |
|------|------|
| **遊戲黃金 (GameGold)** | `monsterLevel × 20` |
| 能源骰子 | 10% 機率 +1（心魔怪另計） |
| 黃金骰子 | 2% 機率 +1 |
| 部隊加成 | 個人 **GameGold** 獎勵的 20% 進入部隊遊戲金庫 |

### 7.3 心魔怪 (Demon) 特殊掉落
| 怪物等級 | 重啟骰子掉落 |
|---------|------------|
| Lv 1-3 | +1 骰子 |
| Lv 4-6 | +2 骰子 |
| Lv 7+ | +3 骰子 |
- 5% 機率觸發全服事件：所有玩家各 +1 能源骰子（`global_dice_bonus`）。

### 7.4 死亡懲罰
- 扣除**身上總 GameGold 的 3%**（最低扣 10）。
  > 即：`penalty = max(10, floor(CharacterStats.GameGold × 0.03))`

### 7.5 連續打卡主動技能（Streak Skills）

> **動態解鎖**：`Streak >= 3` 解鎖進階技能；`Streak >= 7` 解鎖終極技能。  
> **中斷封印**：某日未完成定課導致 `Streak = 0`，技能立即封印，重新累積後才恢復。  
> **與 DDA 聯動**：精進者（Streak ≥ 7）面對 AI 生成的高難度精英怪時，終極技能是抗衡關鍵；一旦斷卡、大絕被封，面對自己養育的強怪將不堪一擊。

## 十、團隊系統 (Team System)

> 本活動正式名稱：**2026 大無限開運親證班**

### 組織架構

```
2026 大無限開運親證班
├── 大隊 A
│   ├── 小隊 A-1
│   ├── 小隊 A-2
│   └── 小隊 A-3

│   ├── 小隊 B-1
│   ├── 小隊 B-2
```

| 層級 | 說明 | 對應欄位 |

- **資金捐贈**（`transferCoinsToTeam` in `store.ts`）：玩家手動捐贈 `Coins` 給**小隊**金庫。

> 限制：僅限**同小隊**成員，不能自給自受。

## 十二、修為排名系統 (`RankTab`)

### 12.1 雙榜架構

修為榜分為兩個子榜，透過頁內 Tab 切換顯示：

| 子榜 | 排序依據 | 資料來源 | Tab 顏色 |
|------|---------|----------|----------|
| 個人榜 | 個人 `Exp` 降序 | `CharacterStats` | 橘色 |
| 小隊榜 | 小隊成員 `Exp` 總和降序 | 對 `leaderboard` 做前端分組 | 藍色 |

> 兩個榜共用同一份 `leaderboard` 資料（從 Supabase `CharacterStats` 撈取全員），無需額外 API。

### 12.2 個人榜設計

- 依全員 `Exp` 降序排列
- **前三名** 顯示金 / 銀 / 銅徽章
- **自身高亮**：當 `currentUserId === p.UserID` 時，列項套用橘色邊框＋淡底色，名字後顯示 🔥
- 每列顯示：名次 ｜ 角色頭像（取 `ROLE_CURE_MAP` 的 `avatar` 表情）｜ 姓名 ｜ 角色 · 小隊 ｜ 修為

### 12.3 小隊榜設計

#### 分組邏輯

```ts
group key = SquadName ?? `__solo_${UserID}`
```

- 有 `SquadName` → 歸入同隊
- 無 `SquadName`（未指派）→ 獨立顯示（key 以 `__solo_` 為前綴）

#### 計算欄位

| 欄位 | 說明 |
|------|------|
| `totalExp` | 隊內所有成員 `Exp` 加總 |
| `memberCount` | 隊員人數 |
| `avgExp` | `totalExp ÷ memberCount`（四捨五入，UI 顯示用） |
| `topMember` | 隊內 `Exp` 最高者（頭像代表隊） |

#### 顯示格式

```
┌──────────────────────────────────────────────┐
│  🥇  [topMember頭像]  小隊名稱                │
│             N 人 · 均 X,XXX 修為 · TeamName  │  
│                              X,XXX 總修為     │
│  成員標籤列：[頭像 姓名 修為] ...             │
└──────────────────────────────────────────────┘
```

- 成員標籤依個人 `Exp` 降序排列
- 前三名小隊整列套用 `bg-white/5` 底色突顯

### 12.4 元件規格

**檔案**：`components/Tabs/RankTab.tsx`

**Props**

| Prop | 型別 | 說明 |
|------|------|------|
| `leaderboard` | `CharacterStats[]` | 全員資料（由 page.tsx 從 Supabase 撈取） |
| `currentUserId` | `string?` | 登入者 UserID，用於自身高亮 |

**內部資料型別**

```ts
interface SquadRankEntry {
    squadName: string;
    teamName?: string;
    totalExp: number;
    memberCount: number;
    members: CharacterStats[];
    topMember: CharacterStats;  // 隊內最高修為成員
}
```

**相依常數**：`ROLE_CURE_MAP`（取 `avatar` 欄位顯示角色表情頭像）
